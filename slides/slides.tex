\begin{frame}{Ideia central}
    \Large
    Queremos usar \textbf{verifica\c{c}\~ao formal} para encontrar vulnerabilidades em programas C usados em IoT.

    \vspace{1em}
    \normalsize
    O caso escolhido \'{e} o \textbf{Eclipse Mosquitto}, um broker MQTT escrito em C.

    \vspace{0.8em}
    \begin{block}{Tese do trabalho}
        Estender o ESBMC com \emph{operational models} para APIs de rede e concorr\^encia, permitindo verificar software MQTT real.
    \end{block}
\end{frame}

\begin{frame}{Por que Mosquitto e MQTT?}
    \begin{itemize}
        \item MQTT \'{e} um protocolo leve muito usado em Internet das Coisas.
        \item O broker \'{e} o ponto central: recebe, roteia e entrega mensagens.
        \item O Mosquitto \'{e} open source, amplamente usado e escrito em C.
        \item Bugs no broker podem afetar toda a rede de dispositivos.
    \end{itemize}

    \vspace{0.8em}
    \alert{Esse contexto junta seguran\c{c}a, C, IoT e verifica\c{c}\~ao formal em um problema real.}
\end{frame}

\begin{frame}{Problema}
    O ESBMC j\'a verifica propriedades importantes em C:

    \begin{itemize}
        \item buffer overflow;
        \item integer overflow;
        \item null pointer dereference;
        \item use-after-free;
        \item memory leak.
    \end{itemize}

    \vspace{0.5em}
    \textbf{Mas software de rede usa muitas APIs externas:}
    \texttt{socket}, \texttt{accept}, \texttt{recv}, \texttt{send}, \texttt{select}, \texttt{poll}, \texttt{pthread}.

    \vspace{0.5em}
    \alert{Sem modelos adequados dessas APIs, a verifica\c{c}\~ao de programas reais fica limitada.}
\end{frame}

\begin{frame}{O que s\~ao \emph{operational models}?}
    \textbf{Operational model} \'{e} uma implementa\c{c}\~ao abstrata de uma fun\c{c}\~ao externa.

    \vspace{0.6em}
    Em vez de verificar a implementa\c{c}\~ao real do sistema operacional, o ESBMC usa um modelo menor e controlado.

    \vspace{0.8em}
    \begin{exampleblock}{Exemplo}
        Um modelo de \texttt{recv()} pode dizer: ``o buffer recebe at\'e \texttt{len} bytes n\~ao determin\'isticos''.
    \end{exampleblock}

    \vspace{0.6em}
    Isso preserva o comportamento relevante para seguran\c{c}a sem explodir o espa\c{c}o de estados.
\end{frame}

\begin{frame}{Pergunta de pesquisa}
    \begin{block}{Pergunta}
        Como estender o ESBMC com \emph{operational models} para APIs de rede e concorr\^encia, de forma a detectar vulnerabilidades em brokers MQTT escritos em C?
    \end{block}

    \vspace{0.8em}
    \textbf{Objetivo geral}

    Desenvolver e avaliar modelos operacionais para permitir a an\'alise formal do Mosquitto com ESBMC.
\end{frame}
